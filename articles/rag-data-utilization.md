## RAGとは？ ChatGPTとの決定的な違い

RAG（Retrieval-Augmented Generation）は、**「外部データを検索してから、その情報を使ってAIが回答を生成する」技術**です。

### 従来のChatGPT（LLMのみ）の限界

通常のChatGPTやClaude、Geminiなどの大規模言語モデル（LLM）は、**学習時点までの一般知識**しか持ちません。

- ❌ 社内の最新マニュアル → 知らない
- ❌ 自社商品の詳細仕様 → 知らない
- ❌ 過去の案件データ → 知らない
- ❌ 顧客対応の履歴 → 知らない

つまり、**自社特有の情報には一切答えられない**のです。

### RAGが解決すること

RAGを導入すると、次のフローが実現します：

1. **ユーザーが質問**（例：「2025年第4四半期の売上レポートを要約して」）
2. **システムが社内データベースを検索**（該当する報告書PDFを自動取得）
3. **検索結果をLLMに渡して回答生成**（「2025年Q4の売上は前年比12%増…」）

→ **社内データをリアルタイムに活用した回答が可能に**

しかも、**LLMを再学習（ファインチューニング）する必要がない**ため、コストも時間も大幅に削減できます。

## RAGのメリット：なぜ企業が今、導入しているのか

### 1. **社内ナレッジの即座な活用**

- 過去の議事録、マニュアル、報告書を瞬時に検索
- 新人でもベテラン並みの回答が可能
- 属人化した知識の共有・継承

### 2. **最新情報への対応**

- データベースを更新すれば、AIも即座に最新情報で回答
- LLMの再学習は不要

### 3. **低コストでの運用**

- GPT-4やClaude 3.5 SonnetなどのAPI利用で十分
- ファインチューニング（数百万円〜）は不要
- 中小企業でも月数千円〜数万円で運用可能

### 4. **情報の透明性**

- 回答の根拠（引用元）を明示できる
- 「どのドキュメントを参照したか」が追跡可能
- ハルシネーション（誤情報生成）のリスク低減

## RAG構築の基本構成：3つのコア技術

RAGシステムは、大きく分けて3つの要素で構成されます。

### 1. **ベクトルデータベース（Vector DB）**

社内データを「ベクトル（数値の配列）」に変換して保存するデータベース。

**代表的なツール：**
- Pinecone（クラウド型、初心者向け）
- Weaviate（オープンソース、柔軟性高）
- Qdrant（高速、Rust製）
- Chroma（ローカル開発に最適）

### 2. **埋め込みモデル（Embedding Model）**

テキストをベクトルに変換するAIモデル。

**主な選択肢：**
- OpenAI Embeddings（text-embedding-3-small/large）
- Voyage AI（精度重視）
- Cohere Embed（多言語対応）
- オープンソース（sentence-transformers等）

### 3. **生成モデル（LLM）**

検索結果を元に回答を生成するAI。

**推奨モデル：**
- GPT-4 Turbo / GPT-4o（バランス型）
- Claude 3.5 Sonnet（長文処理に強い）
- Gemini 1.5 Pro（大量データ対応）

## 実際の構築フロー：5ステップで理解する

ここからが本題です。RAGを実際に構築する際の基本フローを5つのステップで解説します。

### Step 1: データの準備と前処理

まず、AIに読ませたい社内データを準備します。

**対応可能なデータ形式：**
- PDF（マニュアル、報告書）
- Word/Excel（社内文書）
- Markdown/テキスト（議事録、Notion等）
- Webページ（社内Wiki、製品ページ）

**前処理のポイント：**
- 不要な情報（ヘッダー、フッター）の除去
- チャンク分割（1000〜2000トークン単位）
- メタデータの付与（作成日、部署、カテゴリ等）

### Step 2: ベクトル化とDB格納

準備したデータを埋め込みモデルでベクトル化し、Vector DBに保存します。

```python
# 疑似コード（実装例）
from openai import OpenAI
import pinecone

client = OpenAI()
pinecone.init(api_key="YOUR_KEY")

# テキストをベクトル化
embedding = client.embeddings.create(
    input="社内マニュアルのテキスト",
    model="text-embedding-3-small"
)

# Vector DBに保存
index.upsert(vectors=[
    ("doc1", embedding.data[0].embedding, {"source": "manual.pdf"})
])
```

### Step 3: 検索システムの構築

ユーザーの質問を受け取り、関連するドキュメントを検索します。
